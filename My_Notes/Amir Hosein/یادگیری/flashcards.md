HTTP چیست؟ توضیح کامل بده.

📌 تعریف:
HTTP (HyperText Transfer Protocol) پروتکل اصلی ارتباط در وب است که نحوه ارسال و دریافت اطلاعات بین کلاینت (مرورگر) و سرور را مشخص می‌کند.

🔧 نحوه کار:
۱. کلاینت (شما) آدرس سایت را وارد می‌کنید
۲. درخواست HTTP به سرور ارسال می‌شود
۳. سرور درخواست را پردازش می‌کند
۴. پاسخ HTTP حاوی صفحه وب برمی‌گردد
۵. مرورگر پاسخ را نمایش می‌دهد

📋 ویژگی‌ها:
• Stateless: هر درخواست مستقل است، حافظه قبلی ندارد
• متنی (Text-based): خوانا برای انسان
• درخواست-پاسخ: یک درخواست = یک پاسخ

🌐 کاربرد:
بارگذاری صفحات وب، ارسال فرم‌ها، دانلود فایل، APIها

⚠️ نقطه ضعف:
داده‌ها رمزنگاری نمی‌شوند - قابل شنود توسط هکرها

تفاوت کامل HTTP و HTTPS را توضیح بده.

📌 HTTPS (HTTP Secure) = HTTP + لایه امنیتی SSL/TLS

🔐 تفاوت‌های کلیدی:

┌─────────────────┬──────────────────┬──────────────────┐
│     ویژگی      │      HTTP        │     HTTPS        │
├─────────────────┼──────────────────┼──────────────────┤
│   امنیت        │    ❌ ندارد      │    ✅ رمزنگاری   │
│   پورت         │      80          │      443         │
│   گواهینامه    │    ❌ ندارد      │    ✅ SSL needed │
│   سرعت         │    ⚡ سریع‌تر    │    🐢 کمی کندتر  │
│   SEO گوگل     │    ⬇️ ضعیف‌تر   │    ⬆️ بهتر       │
└─────────────────┴──────────────────┴──────────────────┘

🛡️ مزایای HTTPS:
• رمزنگاری end-to-end: داده‌ها قابل خواندن نیستند
• احراز هویت: مطمئن می‌شوید سایت واقعی است
• یکپارچگی داده: تغییر در مسیر قابل تشخیص است

🔑 نحوه کار:
قبل از ارسال HTTP معمولی، یک دست‌دهی امن (Handshake) انجام می‌شود و کلید رمزنگاری تبادل می‌گردد.

💡 نکته:
همیشه به دنبال 🔒 در نوار آدرس باشید!

ساختار یک درخواست HTTP و پاسخ آن را کامل توضیح بده.

📤 ساختار درخواست (Request):

۱. خط شروع (Start Line):
   METHOD URL HTTP-VERSION
   مثال: GET /index.html HTTP/1.1

۲. هدرها (Headers):
   Host: www.example.com
   User-Agent: Mozilla/5.0
   Accept: text/html
   (اطلاعات اضافی درباره درخواست)

۳. بدنه (Body) - اختیاری:
   داده‌های ارسالی مثل اطلاعات فرم

📥 ساختار پاسخ (Response):

۱. خط وضعیت (Status Line):
   HTTP-VERSION STATUS-CODE MESSAGE
   مثال: HTTP/1.1 200 OK

۲. هدرها (Headers):
   Content-Type: text/html
   Content-Length: 1234
   Date: Mon, 23 May 2024
   (اطلاعات درباره پاسخ)

۳. بدنه (Body):
   محتوای واقعی صفحه (HTML, JSON, تصویر...)

🔢 کدهای وضعیت مهم:
• 200 OK - موفق
• 404 Not Found - یافت نشد
• 500 Internal Server Error - خطای سرور
• 301 Moved Permanently - انتقال دائمی

متدهای (Methods) اصلی HTTP را توضیح کامل بده.

📌 متد = فعل یا عملی که می‌خواهید انجام دهید

🛠️ متدهای اصلی:

┌──────────┬────────────────────────────────────────┐
│   GET    │ دریافت اطلاعات از سرور                │
│          │ ➤ فقط خواندن، بدون تغییر داده        │
│          │ ➤ قابل Bookmark، قابل Cache          │
├──────────┼────────────────────────────────────────┤
│   POST   │ ارسال داده جدید به سرور               │
│          │ ➤ ایجاد منبع جدید                    │
│          │ ➤ داده در Body مخفی‌تر است           │
├──────────┼────────────────────────────────────────┤
│   PUT    │ بروزرسانی کامل منبع موجود             │
│          │ ➤ جایگزینی کل داده‌های قبلی          │
├──────────┼────────────────────────────────────────┤
│  PATCH   │ بروزرسانی جزئی منبع                   │
│          │ ➤ تغییر فقط فیلدهای خاص              │
├──────────┼────────────────────────────────────────┤
│ DELETE   │ حذف منبع از سرور                      │
│          │ ➤ حذف دائمی (معمولاً)                │
├──────────┼────────────────────────────────────────┤
│  HEAD    │ دریافت فقط هدرها (بدون بدنه)          │
│          │ ➤ بررسی وجود فایل یا تغییرات         │
└──────────┴────────────────────────────────────────┘

💡 مثال عملی:
فرم ثبت‌نام ➜ POST
مشاهده پروفایل ➜ GET
ویرایش ایمیل ➜ PATCH یا PUT
حذف اکانت ➜ DELETE

مهم‌ترین هدرهای (Headers) HTTP را توضیح بده.

📌 هدر = اطلاعات متا (Metadata) درباره درخواست/پاسخ

🔷 هدرهای درخواست (Request Headers):

Host: www.example.com
➤ نام دامنه سایت مقصد

User-Agent: Mozilla/5.0 (Windows NT 10.0)
➤ نوع مرورگر و سیستم عامل شما

Accept: text/html, application/json
➤ فرمت‌هایی که مرورگر می‌فهمد

Authorization: Bearer token123
➤ اطلاعات لاگین و احراز هویت

Cookie: sessionid=abc123
➤ اطلاعات ذخیره شده از بازدید قبلی

Content-Type: application/x-www-form-urlencoded
➤ نوع داده ارسالی در Body

🔶 هدرهای پاسخ (Response Headers):

Content-Type: text/html; charset=UTF-8
➤ نوع محتوای ارسالی (HTML, JSON, تصویر...)

Content-Length: 2048
➤ حجم داده به بایت

Set-Cookie: sessionid=xyz789
➤ ذخیره اطلاعات در مرورگر کاربر

Cache-Control: max-age=3600
➤ مدت زمان نگهداری در حافظه موقت

Location: /new-page.html
➤ آدرس جدید برای ریدایرکت (کد 301/302)

Server: nginx/1.18.0
➤ نرم‌افزار سرور (امنیتی: معمولاً مخفی می‌کنند)
